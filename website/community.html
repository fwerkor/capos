High-impact contribution areas

  • Hardware compatibility and boot validation
  • WebDesktop UX and management workflows
  • Documentation quality and onboarding paths
  • Security hardening and performance tuning

Collaboration tip

Open a discussion or issue before major implementation work so direction and scope are clear before coding begins.

How to contribute effectively

1. Define problem context

Describe expected behavior, actual behavior, environment details, and reproduction steps clearly.

2. Align proposal

For major features, share design direction first through discussion or issue before implementation.

3. Keep changes focused

Submit small, reviewable pull requests with clear commit messages and concise test evidence.

4. Improve docs together

Documentation, examples, and troubleshooting notes are high-value contributions for every release cycle.

New to open-source collaboration?

Start with typo fixes, onboarding improvements, or issue triage. Small contributions are the fastest way to learn project conventions.
